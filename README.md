# Arduino--
基于Arduino开发板的超声波定位系统
方案二

一、整套 8cm 双超声模拟卫星定位方案 优点

1)多重防抖，适配小物体
2)采用 5 次测距取均值，过滤超声波弱回波带来的跳数；延长 pulseIn 超时时间，更容易捕捉小件物体反射信号。
多层容错，杜绝屏幕乱码、负坐标
3)根号内负数保护，避免程序计算异常；
4)坐标强制裁剪 0~8cm 区间，彻底消除 X 负几十的问题；
5)单路超声失效时借用另一路距离估算，不会直接显示Out Of Area；
6)距离硬阈值 2~80cm，过滤盲区与超大溢出值，根除近距离cmm乱码。
7)显示简洁适配两行 LCD1604
第一行输出坐标 X/Y，第二行输出两路原始测距 dA/dB，信息直观，不用 4 行冗余文字。
模拟卫星定位原理清晰
用三角测距复现 GPS “多观测点解算二维坐标” 逻辑，适合课程演示、原理实验。
硬件层面
成本极低，仅 2 个 HC-SR04，一块 LCD，无昂贵传感器；
接线简单，仅占用少量数字引脚，预留拓展 IO；
安装结构简单，只需要将两个超声固定在正方形单条边两端即可搭建。
二、整套方案明显缺点
1. 几何结构先天缺陷（最核心短板）
1）基线仅 8cm，定位精度随距离衰减极快
物体离探头越远，微小测距误差会被公式放大，几十厘米外 X、Y 数值剧烈抖动，几乎失去参考意义。
2）有效探测区域不是正方形，是窄三角形
水平 X 只能在 0~8cm 之间，物体跑到两个探头左右外侧，数学无解，只能强制钳位边界；纵深 Y 虽理论可达 80cm，但远距离精度报废。
3）两点定位存在位置歧义
同一组 dA、dB 会对应前后两个对称位置，程序无法区分物体是靠近探头还是远处，只能依靠 Y 值粗略判断。
2. HC-SR04 超声波硬件固有问题，对小物体不友好
1）声波呈 15° 锥形发散，波束宽
小件物体很难同时反射两路声波，经常单路丢失回波，造成坐标计算失真；
2）反射依赖硬质平面，软质、细小物体回波微弱，容易测距溢出；
3）存在 2cm 测量盲区，物体贴探头直接失效。
3. 功能局限性
1）仅二维平面定位，无法测量高度 Z 轴；
2）无自动校准功能，两个超声安装间距一旦变动，坐标全部出错，必须手动修改代码SquareSide；
3）环境干扰大：气流、墙壁、桌面反光、其他障碍物都会干扰声波，数值持续漂移；
4）无滤波硬件，仅靠软件平均降噪，抖动只能缓解无法根除。
4. 代码与使用限制
1）单路探头无回波时采用补偿估算，远距离估算坐标误差很大；
2）阈值固定 2~80cm，超出范围直接判定无效物体，远距离实验受限；
3）仅能实现静态物体检测，移动物体坐标滞后、延迟明显（500ms 刷新周期）。
